# plan-harness local-proxy

MCP server + local dashboard for the plan-harness plugin.

## For end users

Nothing to do. The plugin ships with a pre-built bundle at `dist/index.js`. Claude Code launches it via `start.js` and runs instantly — no `npm install`, no network, no build step required.

## For plugin maintainers

Source lives in `src/`. The committed `dist/index.js` is a bundled artifact generated by esbuild.

```bash
cd plan-harness/local-proxy
npm install        # installs devDeps (esbuild, @modelcontextprotocol/sdk) — local only
npm run build      # regenerates dist/index.js
```

**Rules:**

- `dist/index.js` **must be committed** alongside source changes. CI/review should flag PRs that modify `src/` without also updating `dist/`.
- `@modelcontextprotocol/sdk` is a `devDependency` — it is bundled into `dist/`, not resolved at runtime.
- Only `node:*` built-ins are imported at runtime after bundling. No user-facing network or filesystem needs beyond reading/writing plan files.

## Layout

```
local-proxy/
  start.js              — launcher (imports dist/index.js)
  src/                  — source
    index.js            — MCP server + tool registrations
    plan-manager.js     — filesystem CRUD for scenarios
    web-server.js       — local HTTP dashboard
    templates/base.js   — self-contained HTML templates
  dist/
    index.js            — built bundle (committed)
  package.json
```

## Runtime

- Node ≥ 18 required (esbuild target=node18).
- Logs go to **stderr** only (stdout is reserved for MCP JSON-RPC).
- Dashboard listens on `127.0.0.1` by default; exposed externally only via `plan_share` devtunnel.
